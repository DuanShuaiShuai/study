## 发展简史
### 四个阶段
- 1946 - 1957 电子管计算机
    - 二战时期 英国为了破解德国的密文而建造的第一台艾尼阿克（ENIAC）计算机
    - 集成度小 空间占用大
    - 功耗高 运行速度满
    - 操作复杂 更换程序需要接线

- 1957 - 1964 晶体管计算机
    - 贝尔实验室发明了晶体管
    - 集成相对较高 空间占用较小
    - 功耗相对较低 运行速度较快
    - 操作相对简单 交互相对方便
- 1964 - 1980 集成电路计算机
    - 德州仪器的工程师发明了集成电路（IC）
    - 计算机变得更小
    - 功耗变的更低
    - 计算速度变得更快
    - 计算机具备了进入千家万户的条件
    - IBM出了两款计算机
        - 7094  1401
            - 主打的功能不一样
            - 相互无法兼容
            - 不愿意投入两组人力
        - IBM 推出了兼容的产品 System/360 ,这是操作系统的原型
- 1980 - now  超大规模集成电路计算机
    - 一个芯片集成了上百万的晶体管
    - 速度更快 体积更小 价格更低 更能被大众接受
    - 用途丰富 文本处理/表格处理/高交互的游戏与应用
### 微型
#### 阶段
- 1971-1973 500KHz 频率的微型计算机（字长8位）
- 1973-1978 高于1MHz频率的微型计算机（字长8位）
- 1978-1985 500MHz频率的微型计算机（字长16位）
- 1985-2000 高于1GHz频率的微型计算机（字长32位）

- 20世纪 集成/芯片的发展符合摩尔定律，集成电路的性能，每18-24个月就会提升一倍 ，但是21世纪这个定律失效，因为集成电路复杂度越高，散热问题更难解决，所以转向多核Cpu
- 多核Cpu
    - 2005 Intel 奔腾系列多核CPU/AMD速龙系列
    - 2006 intel酷睿四核CPU
    - intel 酷睿系列16核CPU
    - intel 志强系列56核CPU

## 体系结构
### 冯诺依曼体系
- 将程序指令和数据一起存储的计算机涉及概念结构
- 起因
    - 早期的计算机仅含有固定用途的程序
    - 改变程序需要更改结构和重新设计电路
    - 坑爹啊 不能先打会游戏然后在写会代码
    - 把程序存储起来并设计通用电路
- 条件
    - 必须有一个存储器：能够长期记忆程序、数据、中间结构及最终运算结构的能力
    - 必须有一个控制器：能够具备算术、逻辑运算和数据传送等数据加工处理的能力
    - 必须有一个计算器：能够具备算术、逻辑运算和数据传送等数据加工处理的能力
    - 必须有输入设备：能够把需要的程序和数据送至计算机中
    - 必须有输出设备：能够按照要求将处理的结构输出给用户
    - CPU = 计算器 + 控制器 
- 瓶颈 
    - 存储器与cpu是分开的，经常导致cpu 空转，计算快，传输慢 
###  现在计算机的机构
    - 现在计算机在冯诺依曼体系结构基础上进行修改
    - 解决了CPU与存储设备之间的性能差异问题
    - CPU = 计算器 + 控制器 + 存储器
## 层次与编程语言
### 程序翻译与程序解释
    - 程序翻译与程序解释：将人类语言转化成计算机语言 需要进行语言之间的转换
        - 较为高级的计算机语言L1
            - 用L1进行程序逻辑描述
            - L1 生成 L0 这个过程是编译器完成的,就是程序翻译
        - 较为低级的计算机语言L0（程序实际执行的语言）
            - 用L1进行程序逻辑描述
            - L1做为L0输入 这个过程是解释器完成的 是程序解释
    - 计算机执行的指令都是L0
    - 翻译过程生成新的L0程序  解释过程不生成新的L0程序
    - 解释过程由L0编写的解释器去解释L1程序
    - 程序翻译； C/C++  Object-c  Golang 
    - 程序解释：Python php javascript
    - 翻译+解释：java C#
        -java 先编译JVM字节码  然后字节码在解释成机器码
#### 网上总结的
    - 编译程序是一个语言处理程序，它把一个高级语言程序翻译成某个机器的汇编或二进制代码程序，这个二进制代码程序在机器上运行以生成结果。因此通过编译程序使得我们可以先准备好一个在该机器上运行的程序，然后这个程序便会以机器的速度运行。但是在不把整个程序全都翻译结束之后，这个程序是不能开始运行，也不能产生任何结果的。编译和运行是两个独立分开的阶段。但在一个交互环境中，不需要将这两个阶段分隔开，编译就不如解释的方法更方便。这里我们介绍另一种语言处理程序，叫解释程序，它不需要在运行前先把源程序翻译成目标代码，也可以让我们实现在某台机器上运行程序并生成结果。
    - 解释程序接受某个语言的程序并立即运行这个源程序。它的工作模式是一个个的获取、分析并执行源程序语句，一旦第一个语句分析结束，源程序便开始运行并生成结果，它特别适合程序员交互方式的工作情况，即希望在获取下一个语句之前了解每个语句的执行结果，允许执行时修改程序。
    - 编译程序先生成目标代码，后运行，于是程序的运行速度就取决于机器本身的速度；而解释程序一边分析一边运行，机器本身再快也会比较慢，想让程序跑得飞快就不能用解释程序

## 计算单位
### 容量单位
- bit(比特位)
    - 在物理层面，高低电平记录信息
    - 理论上只认识0/1两种状态 
    - 0/1称为bit(比特位)
    - 0和1表示的内容太小了 需要更大的容量表示方法
- byte（字节）
    - 1Byte = 8 bits
- Kb
    - 1KB = 1024B
- MB
    - 1MB = 1024KB
- 为什么网上买的移动硬盘500G 而实际只有465G？
    - 硬盘商一般用10进制标记容量
    - 500*1000*1000*1000/1024/1024/1024=465
### 速度单位
- 网路的速度单位
    - 100M宽带 100M/S
    - 网络常用的单位为（Mbps）
    - 100M/S = 100Mbps =100Mbit/s=(100/8)MB/s=12.5MB/s
- CPU 的速度单位
    - CPU的速度一般体现为CPU的时钟频率
    - CPU的时钟频率的单位一般是赫兹（HZ）
    - 主流CPU的时钟频率都在2Ghz以上
    - Hz其实就是秒分之一 它是每秒钟的中周期性变动的重复次数的计量
    - 2GHZ = 每秒20亿次
        - 代表 cpu 高低电平每秒变化了20亿次
## 字符与编码集
### 字符编码集的历史
- ASCLL码
    - 使用了7个bits就可以完全表示ASCLL码
    - 包含95个可打印字符
    - 33个不可打印字符（包含控制字符）
    - 33+95=128=2*2*2*2*2*2*2
    - ASCLL码表 https://baike.baidu.com/item/ASCII/309296?fr=aladdin
    - 很多应用或者国家中的符号都无法表示(extend Ascll) 
        - 第一次最ASCLL码进行扩充 7bits=>8bits(256)
    - 字符编码的国际化
        - 欧洲、中亚、东亚、拉丁美洲国家的语言的多样性
        - 语言体系不一样 不以有限字符组合的语言
        - 中国、韩国、日本等语言最为复杂
### 中文编码
- GB2312
    - <<信息交换汉子编码字符集---基本集>>
    - 一共收录了7445个字符
    - 包含6763个汉字和682个其他符号
    - 但是不符合国际的标准
- GBK
    - 《汉字内码扩展规范》
    - 向下兼容了GB2312,向上支持国际ISO标准
    - 收录了21003个个汉字，支持全部中日韩汉字
- GBK GB2312  但是只是一个本地的编码 外国人的电脑如果没有安装这种，会出现乱码
- Unicode:兼容全球的字符集
    - 统一码、万国码、单一码
    - Unicode定义了世界通用的符号集 ，UTF-*实现了编码
    - UTF-8 以字节为单位对Unicode进行编码
    - 中国的window系统默认使用GBK编码
    - 编程推荐使用UTF-8编码
